# Shop31 — структура проєкту (довідник)

Коротко: **frontend** (React + Vite) у корені репозиторію; **сервер оплати** Stripe — у каталозі `server/`. Дані замовлень, бонусів, кошика тощо в демо зберігаються в **localStorage** у браузері.

---

## Корінь репозиторію

| Файл / папка | Призначення |
|--------------|-------------|
| `package.json` | Залежності й скрипти фронтенду (`dev`, `build`, `preview`). |
| `vite.config.ts` | Збірка Vite, base path, проксі `/api` → payment server, плагін для `dist/index.html`. |
| `index.html` | Точка входу HTML для Vite (коментар у файлі вказує на цей довідник). |
| `src/index.css` | Глобальні стилі та CSS-змінні теми. |
| `src/App.css` | Стилі компонентів і сторінок (підключається з App.tsx). |
| `src/` | Увесь код інтерфейсу магазину та адмінки. |
| `server/` | Express + Stripe: PaymentIntent, не зберігає секрети фронту. |
| `.env` / `.env.example` | Змінні для Vite (публічні ключі). Не комітити секрети. |
| `eslint.config.js` | ESLint 9: TS + React Hooks + react-refresh; `npm run lint`. |
| `tsconfig.json` | Корінь solution: посилання на `tsconfig.app` і `tsconfig.node`. |
| `tsconfig.app.json` | TypeScript для коду в `src/` (браузер, React). |
| `tsconfig.node.json` | TypeScript лише для `vite.config.ts` (Node). |
| `dist/` | Результат `npm run build`; не редагувати вручну. |
| `node_modules/` | Залежності npm; не ваш код. |

## `public/` — статика без збірки

| Файл | Призначення |
|------|-------------|
| `favicon.svg` | Іконка вкладки; шлях у `index.html`. |
| `icons.svg` | Допоміжні SVG-спрайти, якщо підключені з HTML/CSS. |

Файли з `public/` копіюються в корінь `dist/` під тим самим шляхом (на відмінність від імпортів з `src/assets`).

## `src/assets/` — ресурси, що йдуть через Vite

| Що тут | Призначення |
|--------|-------------|
| `hero.png`, інші зображення | Використовуються сторінками та `data/products` через `import` — Vite дає хешовані URL у `dist/assets/`. |
| `products/**` | Фото товарів; імпортуються з `data/products.ts`. |
| `react.svg` | Типовий зразок Vite; може не використовуватись у вітрині — перевірте імпорти перед видаленням. |

## `src/` — шари

### Точка входу та маршрути

| Файл | Що робить |
|------|-----------|
| `main.tsx` | Монтує React у `#root`, підключає `index.css` і `App`. |
| `App.tsx` | `BrowserRouter` / `HashRouter`, усі маршрути покупця й `/admin/*`, провайдери `I18n`, `Auth`, `Shop`. |

### Контекст і стан

| Файл | Що робить |
|------|-----------|
| `context/AuthContext.tsx` | Користувач, вхід/вихід, реєстрація; синхронізація з `auth/storage`. |
| `context/ShopContext.tsx` | Кошик, обране, додавання товарів; читає/пише `shop/storage`. |

### Інтернаціоналізація

| Файл | Що робить |
|------|-----------|
| `i18n/types.ts` | Тип `Locale`, ключ localStorage. |
| `i18n/strings.ts` | Словник рядків UK/EN, функція `translate`. |
| `i18n/I18nContext.tsx` | `I18nProvider`, `useI18n`, `t()`, зміна `document.lang`, виклик `setNumberFormatLocale`. |

### Дані каталогу (статичні + злиття з адмінкою)

| Файл | Що робить |

## `server/`

| Файл | Роль |
|------|------|
| `package.json` | Залежності Express/Stripe; скрипт `start` → `node index.mjs`. |
| `index.mjs` | API: створення PaymentIntent, підтвердження; Stripe Connect опційно. |
| `env.example` | Приклад змінних для `server/.env` (секретний ключ Stripe — лише тут, не у Vite). |

**Зв’язок фронт ↔ сервер:** `src/config/stripeCheckout.ts` (URL API, publishable key) + `vite.config.ts` проксі `/api` у режимі dev; після збірки фронту URL API задається змінною `VITE_PAYMENT_API_URL`, якщо потрібен інший хост.

У вихідному коді майже кожного модуля в `src/` (та в `vite.config.ts`, `server/index.mjs`) на початку файлу є блок-коментар **«Shop31 — …»** з коротким описом і рядком **«Зв’язки: …»** до пов’язаних файлів. HTML/CSS мають короткі коментарі з посиланням на цей файл. Конфіги ESLint/TS і `package.json` мають короткі підписи призначення.
